# 과제 1. 컴퓨터 하드웨어 구조

1.  CPU의 구성요소 ALU, CU, 그리고 레지스터 각각의 역할을 간략하게 정리해보자.
    > - ALU (Arithmetic Ligit Unit)   
        산술연산과 논리연산을 수행하는 장치.  
        덧셈, 뺄셈, 곱셈, 나눗셈 / AND, OR, NOT  
        레지스터를 통해 피연산자를 받아들이고, CU (제어장치)로부터 수행할 연산을 알려주는 제어 신호를 받아들여 연산 수행.
    > - CU (Control Unit)  
        명령어 해독, 실행, 제어를 담당하는 장치.  
        Control signal이라는 전기 신호를 내보내고 명령어를 해석함.  
    > - Register  
        데이터를 일시적으로 저장하는 장치.  
        프로그램을 실행하는 데 필요한 값들을 임시로 저장.  
        CPU 안에는 여러 개의 레지스터가 존재하고 각각 다른 이름과 역할을 가지고 있음.  
        가장 빠른 속도로 접근 가능한 메모리.  


2.  메인 메모리와 보조기억장치의 차이를 간략하게 서술해보자.
    > - 메인 메모리(주기억장치)  
        CPU가 직접 접근하여 데이터를 읽고 쓸 수 있는 기억장치.  
        RAM(Ranodm Access Memory)과 ROM(Read Only Memory)이 있음.  
    > - 보조기억장치  
        CPU가 직접 접근하여 데이터를 읽고 쓸 수 없는 기억장치.  
        접근하려면 디바이스 드라이버와 시스템 콜을 통하여 기억장치의 특정 위치의 내용을 주기억장치로 로드한 뒤 읽어야 함.
        SSD, HDD가 있음.


3. 버스 시스템은 데이터를 주고받기 위한 경로로, 데이터의 종류에 따라 세 가지로 구분할 수 있다. 세 가지는 무엇인지 말해보자.
    > - 데이터 버스(Data Bus)  
        시스템 컴포넌트 간 처리 데이터를 전달.  
    > - 주소 버스(Address Bus)  
        메모리 주소나 I/O unit 포트번호를 전달.  
    > - 제어 버스(Control Bus)  
        CPU와 기억장치 또는 I/O unit 사이의 제어 신호 전달.  
        Read와 Write 신호 전달.
